# Upgrade Guide — v0.2.0

> **Links:**
> - CHANGELOG: [v0.2.0](../CHANGELOG.md#020--2026-04-26)
> - PR breaking changes: none (minor release, pre-1.0.0)

## What's new

- **Phase toggle per project:** set `phases` in `forja/.forja-config.json` (or `~/.forja/config.json`) to enable/disable individual pipeline phases without changing policies or prompts. Each phase has an independent boolean (`dev`, `test`, `perf`, `security`, `review`, `homolog`, `pr`), all `true` by default.
- **`PhasesEnabledSchema` and `PhasesEnabled` type** exported from `src/schemas/config.ts` — now part of the public surface.
- Dashboard design system with black/white/gold palette, Recharts trend charts, heatmap, run comparison, finding drill-down, cost page with budget alerts, and Gantt chart.
- Full-text run search with tsvector + GIN index (migration `0005_search_vector.sql`).
- CLI: `forja help <cmd>`, shell completions (`bash`/`zsh`/`fish`), `--dry-run` global flag, `forja doctor`, `forja config migrate`.
- i18n: `artifact_language` vs `prompt_language` separation; `next-intl` setup with `en` and `pt-BR` catalogs.
- Integration providers: Jira, GitLab, Azure DevOps, Bitbucket, Datadog — all via the pluggable `IntegrationProvider` factory.
- OpenTelemetry tracing with configurable OTLP exporters and W3C TraceContext propagation.
- Hook resilience: `RetryEngine` with exponential backoff + jitter, Dead-Letter Queue persisted in Postgres, circuit breaker per endpoint.
- `SEMVER.md` declaring the public surface and SemVer contract; `DEPRECATIONS.md` with `warnDeprecated` helper.

## Migration steps

1. No action required for existing projects — all new features are opt-in.
2. To use the phase toggle, add a `phases` block to `forja/.forja-config.json`:
   ```json
   {
     "phases": {
       "perf": false,
       "security": false
     }
   }
   ```
   Omitted phases default to `true` (enabled).
3. If you query `ConfigSchema` programmatically, the new `phases` field is optional and backward-compatible — no code changes needed.
4. Run `forja doctor` to validate your environment after upgrading.
