# Upgrade Guide ŌĆö v1.0

> **Links:**
> - CHANGELOG: [v1.0](../CHANGELOG.md#v100)
> - PR breaking changes: https://github.com/livertonoliveira/forja/pulls?q=label%3Abreaking

## What's new

- **Plugin system**: third-party plugins via `forja plugin` DSL ŌĆö install, register hooks, and extend pipeline phases
- **Gates DSL (audits as code)**: define gate policies in YAML/TypeScript, evaluated deterministically with exit codes 0/1/2
- **Schema versioning**: `schema_version` field on all 7 core Postgres tables; migration 0004 adds the column
- **Migration CLI**: `forja migrate` command runs pending migrations with automatic detection and dry-run support
- **Cost tracking**: cache token costs (`cache_creation_input_tokens`, `cache_read_input_tokens`) billed at correct rates (1.25├Ś write, 0.10├Ś read) and stored in DB and JSONL
- **Forja UI (Zagreb)**: Next.js dashboard for run timeline, issue history, cost breakdown, and finding heatmap
- **Pipeline FSM**: explicit finite state machine with Postgres persistence, phase checkpoints, and resume support
- **Replay mode**: `forja replay` reproduces past runs for regression detection
- **Scheduled runs**: `forja schedule` for native cron-style run scheduling
- **SEMVER.md**: public API surface declared and SemVer 1.0 contract established
- **DEPRECATIONS.md**: deprecation policy with `warnDeprecated` runtime helper

## Breaking changes

- `forja infra migrate` is required for existing remote Postgres databases before using v1.0 (migration 0004 adds `schema_version` column)
- Plugin hook timeout is now enforced at 5000 ms (`pluginHookTimeoutMs` in `ConfigSchema`); plugins exceeding this are terminated
- CLI binary renamed to `forja.js` for npm compatibility ŌĆö update any scripts that reference the old binary name
- Phase allowlist is now enforced via `PreToolUse` hook; unauthorized tools are blocked with exit code 2

## Deprecations (scheduled for v1.2)

- `forja infra` subcommands will be merged into `forja migrate` ŌĆö use `forja migrate` directly going forward
- The `--allow-branch` flag in `release.ts` is deprecated; branch validation will be handled by CI

## Migration steps

1. Run `forja migrate` to apply pending migrations (adds `schema_version` to all core tables)
2. Update `forja/config.md` if you have custom `pluginHookTimeoutMs` values ŌĆö the new default is 5000 ms
3. If using remote Postgres, run `forja infra migrate` before upgrading the binary
4. Review `DEPRECATIONS.md` for helper `warnDeprecated` usage in your plugins
5. Update any CI scripts referencing the old binary name to use `forja.js`

## Known issues

- `forja replay` does not support runs created before v0.1.1 (missing span ID metadata)
- The Forja UI requires `npm install --legacy-peer-deps` on first setup due to peer dependency conflicts in Next.js
