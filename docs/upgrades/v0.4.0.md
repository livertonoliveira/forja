# Upgrade Guide — v0.4.0

> **Links:**
> - CHANGELOG: [v0.4.0](../CHANGELOG.md#v040)
> - PRs: #154, #155

## What's new

- Pipeline commands (`forja:develop`, `forja:test`, `forja:perf`, `forja:security`, `forja:review`, `forja:homolog`, `forja:pr`) now automatically read `artifact_language` and enabled pipeline phases from `forja/config.md` and propagate them to every agent invocation — no more manual per-command language flags.
- `forja:run` skips disabled phases based on the `Pipeline Phases` section in `forja/config.md`.
- npm publish now skips gracefully if the version already exists on the registry, preventing CI failures on retried releases.

## Breaking changes

- None. Existing `forja/config.md` files remain valid. Commands fall back to defaults when the new fields are absent.

## Deprecations (scheduled for v0.6.0)

- None.

## Migration steps

1. If you initialized Forja before v0.3.0, re-run `forja:init` (or add `artifact_language` and `Pipeline Phases` manually to `forja/config.md`) so the new propagation logic picks them up.
