# Upgrade Guide — v0.3.0

> **Links:**
> - CHANGELOG: [v0.3.0](../CHANGELOG.md#v030)
> - PR: #153

## What's new

- `forja:init` now asks three interactive questions during setup:
  - **Gate behavior**: strict (block on high/medium) or lenient (block on critical only)
  - **Language**: artifact language, prompt language, and code language (defaults to pt-BR / en-US / en-US)
  - **Pipeline phases**: which phases to enable (develop, test, perf, security, review, homolog)
- `forja/config.md` now includes `artifact_language`, `prompt_language`, and `code_language` fields under Conventions
- `forja/config.md` now includes a `Pipeline Phases` section listing enabled/disabled phases

## Breaking changes

- None. Existing `forja/config.md` files remain valid; new fields are additive.

## Deprecations (scheduled for v0.5.0)

- None.

## Migration steps

1. Re-run `forja:init` in existing projects to populate the new language and pipeline phase fields, or add them manually to `forja/config.md`.
